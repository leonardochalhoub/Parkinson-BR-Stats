# Parkinson BR — Infraestrutura de Neuroimagem

> **Live dashboard →** [leonardochalhoub.github.io/Parkinson-BR-Stats](https://leonardochalhoub.github.io/Parkinson-BR-Stats/)

Distribuição de equipamentos de Ressonância Magnética (RM) no Brasil por UF, setor (SUS vs. Privado) e ano — de janeiro de 2005 a dezembro de 2025 — com base nos microdados mensais do DATASUS/CNES.

The dashboard is a fully static single-page app (zero backend) deployed on GitHub Pages. Data is produced by a local PySpark lakehouse pipeline and exported as a single JSON file.

---

## Overview

Brazil has one of the most unequal distributions of neuroimaging infrastructure in the world. This project quantifies that disparity: how many MRI machines exist per state, how many are available through the public health system (SUS), and how the gap has evolved over 20 years.

The analysis supports ongoing research into barriers to early Parkinson's disease diagnosis in Brazil.

**Key findings (2025 snapshot, avg monthly equipment count):**

| Metric | Value |
|---|---|
| Total MRI equipment (Brazil) | ~10,080 |
| São Paulo alone | ~3,054 (30 %) |
| Top 3 states (SP + RJ + MG) | ~56 % of national total |
| Roraima (lowest) | ~12 |

## Running the pipeline

### 1 — Download raw CNES EQ files from DATASUS FTP

Downloads all `EQ??YYMM.dbc` files (≈ 6,600 files, ~27 states × 246 months). Idempotent — skips files already on disk.

```bash
python app/src/bronze/cnes_eq_download_all_from_source.py
# output: data/cnes_eq/*.dbc
```

### 2 — Bronze: ingest DBC files into Delta Lake

```bash
PYTHONPATH=. python app/src/bronze/cnes_eq.py
# output: lakehouse/bronze/cnes_eq/  (partitioned by estado → ano → mes)
```

### 3 — Silver: build population panel (IBGE SIDRA)

```bash
PYTHONPATH=. python app/src/silver/population.py
# output: lakehouse/silver/population/
```

### 4 — Silver: compute MRI indicators and export JSON

```bash
PYTHONPATH=. python app/src/silver/mri_br.py
# output: lakehouse/silver/mri_br/
#         app/web/data/mri_br_state_year.json  (local dev)
#         docs/data/mri_br_state_year.json     (GitHub Pages)
```

### 5 — Serve the dashboard locally

Any static file server works:

```bash
cd app/web
python -m http.server 8000
# open http://localhost:8000
```

Or with Docker:

```bash
# First run step 4 to generate app/web/data/mri_br_state_year.json
docker build -t parkinson-br app/web
docker run -p 8080:80 parkinson-br
# open http://localhost:8080
```

## Dashboard features

- **Choropleth map** — MRI density by UF, switchable color scales
- **State ranking** — horizontal bar chart, sortable by total / SUS / private
- **Evolução Nacional** — stacked yearly bar (SUS + private) or single series
- **Variação Anual (Δ)** — year-over-year delta with integer labels
- **Evolução por Região** — multi-line time series by macro-region
- **KPI cards** — national totals, per-capita, cumulative and YoY growth
- Filters: metric (total vs. per 1M pop), sector (all / SUS / private), year
- Export: Excel workbook (all data + regional sheet) and PNG ZIP of all charts
- Dark mode toggle

## Data sources

| Source | What | URL |
|---|---|---|
| DATASUS — CNES/EQ | Monthly MRI equipment counts by facility, state, and SUS indicator | [datasus.saude.gov.br](https://datasus.saude.gov.br/transferencia-de-arquivos/) |
| IBGE SIDRA — Tabela 6579 | Resident population estimates by UF (variable 9324) | [sidra.ibge.gov.br/tabela/6579](https://sidra.ibge.gov.br/tabela/6579) |

**Equipment filter:** `CODEQUIP = '42'` (Ressonância Magnética)  
**Sector flag:** `IND_SUS = 1` → available to SUS patients; `IND_SUS = 0` → private only  
**Period:** January 2005 – December 2025 (monthly)

### Methodology note

`total_mri_avg` is the annual average of monthly equipment counts per facility (`QT_EXIST`), then summed across all facilities in the state. Facilities that switch `IND_SUS` during the year are handled correctly: both sector averages share the same monthly denominator so `sus + private = total` exactly.

## Querying the Bronze data directly

```python
from app.src.silver.common import build_delta_spark

spark = build_delta_spark("explore")
df = spark.read.format("delta").load("lakehouse/bronze/cnes_eq")

# MRI count by state, 2025
(df.filter("CODEQUIP = '42' AND ano = 2025")
   .groupBy("estado")
   .agg({"QT_EXIST": "sum"})
   .orderBy("sum(QT_EXIST)", ascending=False)
   .show())
```
